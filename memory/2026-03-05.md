## Session 5 (~6:41 PM) — V13 Item #5: Web Chat Bridge

### V13 Item #5: Web Chat Bridge — DONE ✅

**What:** Dashboard-to-King Mouse VM communication. Customers can talk to their dedicated King Mouse instance directly from the web dashboard, with full agent pipeline (SOUL.md, memory, tools, Owner Logic Engine).

**Architecture:**
1. **`scripts/chat-bridge.mjs`** — Node.js WebSocket relay that runs on VM
   - Connects to local OpenClaw gateway at `ws://127.0.0.1:18789`
   - Handles full WS protocol: connect challenge → connect handshake → chat.send → collect streaming/final events
   - Outputs JSON: `{"reply": "...", "error": null}`
   - Requires Node 22+ (native WebSocket)
   - 2-minute timeout, configurable via env

2. **`/api/vm/chat` (POST)** — Vercel API route that proxies to VM
   - Looks up customer's VM from Supabase `employee_vms` table
   - Runs `node chat-bridge.mjs 'message'` on VM via Orgo `executeBash`
   - Parses JSON output, returns reply
   - Includes billing check (pre-flight balance) + usage recording
   - Falls back gracefully if VM not found or not responding

3. **`/api/vm/chat` (GET)** — VM availability check
   - Frontend calls on mount to determine routing mode
   - Returns `{available, vmId, vmStatus, employeeName}`

4. **Dashboard King Mouse page** — Dual-mode chat UI
   - Auto-detects if customer has an active VM
   - **VM mode**: Routes through `/api/vm/chat` → full agent (green "Live VM" badge)
   - **Fallback mode**: Routes through `/api/chat` → direct LLM (amber "Cloud Mode" badge)
   - Auto-falls back if VM becomes unavailable mid-chat
   - Customer-focused prompts, "King Mouse is thinking..." indicator

5. **Provision update** — Both provision paths (bash script + step-by-step) download chat-bridge.mjs to `$HOME/chat-bridge.mjs` during setup

**Protocol details (OpenClaw Gateway WS):**
- Request: `{"type": "req", "id": "<uuid>", "method": "chat.send", "params": {"message": "...", "sessionKey": "main", "deliver": false}}`
- Flow: connect.challenge → connect (no auth for mode:open) → hello → chat.send → chat events (delta/final)
- Final event: `{"type": "event", "event": "chat", "payload": {"state": "final", "message": {"content": [{"type": "text", "text": "..."}]}}}`
